BRIGHTSTEPS 2.0.1 — EASY GITHUB DEPLOYMENT

You do NOT need to run PowerShell and you do NOT need to upload audio files manually.

1. Extract this ZIP.
2. Open your GitHub repository: willis-ys/brightsteps-learning
3. Upload EVERYTHING inside this folder to the repository root.
   Important: include .github/workflows/deploy-pages.yml
4. Open Settings > Pages.
5. Under Build and deployment, set Source to GitHub Actions.
6. Return to the Actions tab.

The workflow named "Deploy BrightSteps 2.0.1" normally starts automatically after the upload commit.
If it does not, open that workflow and select Run workflow > Run workflow.

A successful run shows two green jobs: build and deploy.
Then open:
https://willis-ys.github.io/brightsteps-learning/

If the workflow is not listed, the .github folder was not uploaded.
